## 🚀 Getting Started

### Prerequisites

```bash
node >= 18.0.0
npm >= 9.0.0
expo-cli >= 6.0.0
iOS Simulator (Mac) OR Android Studio OR Physical Device
```

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/rajstories/KAVACH.git
cd KAVACH/kavach-mobile

# 2. Install dependencies
npm install

# 3. Configure environment
cp .env.example .env
# Edit .env — set EXPO_PUBLIC_API_URL to your backend

# 4. Start the development server
npx expo start

# 5. Scan QR with Expo Go app
#    OR press 'i' for iOS Simulator
#    OR press 'a' for Android Emulator
```

### Environment Variables

```bash
# .env
EXPO_PUBLIC_API_URL=http://localhost:3001        # Backend API
EXPO_PUBLIC_APP_ENV=development                  # development | production
EXPO_PUBLIC_ML_URL=http://localhost:8000         # ML Service
```

### Running the Full Stack

```bash
# Terminal 1 — Backend
cd ../backend && npm run dev

# Terminal 2 — ML Service
cd ../ml-service && uvicorn main:app --reload

# Terminal 3 — Mobile App
cd kavach-mobile && npx expo start
```

## 🔒 Security Architecture

```
AUTHENTICATION FLOW
─────────────────────────────────────────────────────────
User Input → API Validation → JWT Token
                                    ↓
                          Expo SecureStore
                          (hardware-backed encryption)
                                    ↓
                     All API requests → Bearer header
                                    ↓
                     401 detected → Auto logout + redirect

BIOMETRIC FLOW
─────────────────────────────────────────────────────────
App Resume → Check SecureStore for existing token
                                    ↓
           Token exists → Biometric prompt (Face ID/Print)
                                    ↓
           Success → Restore session silently
           Failure  → Show full login form

DATA SECURITY GUARANTEES
─────────────────────────────────────────────────────────
  ✅  All tokens in Expo SecureStore (not AsyncStorage)
  ✅  API calls over HTTPS only — no HTTP fallback
  ✅  No citizen PII stored on device
  ✅  Only @gov.in / @nic.in emails accepted
  ✅  Session timeout: 8 hours automatic
```
